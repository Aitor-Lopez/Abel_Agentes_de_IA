│  días, bloqueo tras 5 intentos fallidos, y prohibición de reutilizar las     │

│  - **Impacto**: Riesgo de suplantación de identidad y acceso no autorizado   │
│  durante procesos de recuperación.                                           │
│  - **Recomendación**: Establecer procedimiento formal de recuperación con    │
│  doble autenticación (ej. correo + SMS) y registro de todas las              │
│  solicitudes.                                                                │

│     → Sin EDR, sin control de dispositivos, sin cifrado de disco.            │

│  ## ANÁLISIS DE RIESGOS PARA EL REGISTRO DE TRATAMIENTOS GDPR                │
│                                                                              │
│  El **Registro de Tratamientos** (`registro tratamiento.json`) contiene      │
│  datos personales de usuarios, empleados y proveedores, incluyendo nombres,  │
│  correos, direcciones IP, y en algunos casos, datos de tarjetas de pago. Se  │
│  identifican los siguientes riesgos:                                         │
│                                                                              │
│  1. **Riesgo Legal**: Falta de base legal válida (Art. 6 RGPD) y ausencia    │
│  de consentimiento documentado (Art. 7).                                     │
│     → *Impacto*: Nulidad del tratamiento, sanciones administrativas (hasta   │
│  4% del volumen de negocios).                                                │
│                                                                              │
│  2. **Riesgo Operativo**: Ausencia de mecanismos para ejercer derechos       │
│  ARCO+ (Art. 15–22).                                                         │
│     → *Impacto*: Reclamaciones ante la AEPD, demandas colectivas, daño       │
│  reputacional.                                                               │
│                                                                              │
│  3. **Riesgo Técnico**: Retención indefinida de datos sin criterios (Art.    │
│  5.1.e).                                                                     │
│     → *Impacto*: Mayor superficie de ataque, riesgo de brechas y multas por  │
│  acumulación innecesaria.                                                    │
│                                                                              │
│  4. **Riesgo Transfronterizo**: Transferencias internacionales sin           │
│  garantías legales (Art. 46).                                                │
│     → *Impacto*: Bloqueo de operaciones, sanciones por transferencia ilegal  │
│  de datos.                                                                   │
│                                                                              │
│  **Mitigación Propuesta**:                                                   │
│  - Priorizar la implementación de los PAC relacionados con base legal,       │
│  derechos ARCO+, EIPD y transferencias internacionales.                      │
│  - Asignar un responsable único (DPO) para supervisar la corrección de       │
│  estos riesgos.                                                              │
│  - Integrar el Registro de Tratamientos con el SGSI y el sistema de gestión  │
│  de riesgos corporativos.                                                    │

│  ## ANÁLISIS DE RIESGOS PARA EL REGISTRO DE TRATAMIENTOS GDPR                │
│                                                                              │
│  El **Registro de Tratamientos** (`registro tratamiento.json`) contiene      │
│  datos personales de usuarios, empleados y proveedores, incluyendo nombres,  │
│  correos, direcciones IP, y en algunos casos, datos de tarjetas de pago. Se  │
│  identifican los siguientes riesgos:                                         │
│                                                                              │
│  1. **Riesgo Legal**: Falta de base legal válida (Art. 6 RGPD) y ausencia    │
│  de consentimiento documentado (Art. 7).                                     │
│     → *Impacto*: Nulidad del tratamiento, sanciones administrativas (hasta   │
│  4% del volumen de negocios).                                                │
│                                                                              │
│  2. **Riesgo Operativo**: Ausencia de mecanismos para ejercer derechos       │
│  ARCO+ (Art. 15–22).                                                         │
│     → *Impacto*: Reclamaciones ante la AEPD, demandas colectivas, daño       │
│  reputacional.                                                               │
│                                                                              │
│  3. **Riesgo Técnico**: Retención indefinida de datos sin criterios (Art.    │
│  5.1.e).                                                                     │
│     → *Impacto*: Mayor superficie de ataque, riesgo de brechas y multas por  │
│  acumulación innecesaria.                                                    │
│                                                                              │
│  4. **Riesgo Transfronterizo**: Transferencias internacionales sin           │
│  garantías legales (Art. 46).                                                │
│     → *Impacto*: Bloqueo de operaciones, sanciones por transferencia ilegal  │
│  de datos.                                                                   │
│                                                                              │
│  **Mitigación Propuesta**:                                                   │
│  - Priorizar la implementación de los PAC relacionados con base legal,       │
│  derechos ARCO+, EIPD y transferencias internacionales.                      │
│  - Asignar un responsable único (DPO) para supervisar la corrección de       │
│  estos riesgos.                                                              │
│  - Integrar el Registro de Tratamientos con el SGSI y el sistema de gestión  │
│  de riesgos corporativos.                                                    │
